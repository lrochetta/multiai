**Sprint :** 🔴 1 — BLOCKERS

**Effort :** 5 min
**Agent assigné :** Sentinel (QA/Sécurité)
**Dépendances :** Aucune

---

## Contexte

Une clé API DeepSeek réelle (`***REMOVED-REVOKED-DEEPSEEK-CREDENTIAL***`) est présente en clair dans :
`brainstorm laurent/clé deepseek ne pas mettre dans le repo.txt`

Le fichier est ignoré par `.gitignore` mais reste sur le disque, synchronisable par backup cloud (OneDrive, iCloud, etc.).

## Ce qu'il faut faire

### Étape 1 — Révoquer l'ancienne clé
1. Aller sur https://platform.deepseek.com → API Keys
2. Trouver la clé `sk-883d82d5...` → **Révoquer**
3. Créer une nouvelle clé API

### Étape 2 — Mettre à jour multiai
```bash
multiai config --provider deepseek
```
Coller la nouvelle clé dans le wizard.

### Étape 3 — Vérifier que ça marche
```bash
multiai launch -p ds
```
Le processus enfant doit se lancer avec la nouvelle clé.

### Étape 4 — Nettoyer le disque
```bash
rm -rf "brainstorm laurent/"
```

### Étape 5 — Vérifier qu'il ne reste rien
```bash
rg -n 'sk-[a-zA-Z0-9]{20,}' --glob '!.git' --glob '!node_modules' --glob '!log' --glob '!dist' --glob '!audit'
```
