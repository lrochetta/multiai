# Story 1.3 — Vérifier la signature Cosign dans l'auto-update

**Sprint :** 🔴 1 — BLOCKERS
**Priorité :** CRITICAL
**Effort :** 2h
**Agent assigné :** Forge (Dev)
**Dépendances :** S1.2 (URL hardcodée avant Cosign)
**Fichier cible :** `multiai-go/internal/update/update.go`

---

## Contexte

Actuellement, `downloadAndVerifyRelease` vérifie uniquement le SHA256 de l'archive contre `checksums.txt`. Mais les deux fichiers sont sur le même serveur → un attaquant peut remplacer les deux.

La release GitHub inclut `checksums.txt.sig` et `checksums.txt.pem` (Cosign keyless). Ces signatures doivent être vérifiées **avant** le SHA256.

Source : Audit sécurité C-2 + Codex adversarial.

## Ce qu'il faut faire

### Étape 1 — Télécharger les 3 fichiers

En plus de l'archive, télécharger :
- `checksums.txt` (existant)
- `checksums.txt.sig` (nouveau)
- `checksums.txt.pem` (nouveau)

### Étape 2 — Vérifier la signature Cosign

```go
func verifyCosignSignature(checksumsPath, sigPath, certPath string) error {
    // Vérifier via os/exec: cosign verify-blob
    cmd := exec.Command("cosign", "verify-blob",
        "--certificate", certPath,
        "--signature", sigPath,
        "--certificate-identity-regexp",
        "https://github.com/lrochetta/multiai/.github/workflows/release.yml@refs/tags/v.*",
        "--certificate-oidc-issuer",
        "https://token.actions.githubusercontent.com",
        checksumsPath,
    )
    return cmd.Run()
}
```

### Étape 3 — Fallback si Cosign absent

```go
if _, err := exec.LookPath("cosign"); err != nil {
    if os.Getenv("MULTIAI_REQUIRE_COSIGN") == "1" {
        return fmt.Errorf("cosign requis mais non trouvé: %w", err)
    }
    fmt.Fprintf(os.Stderr, "[!] cosign non trouvé — la signature ne peut pas être vérifiée\n")
    fmt.Fprintf(os.Stderr, "    Installez cosign: https://docs.sigstore.dev/cosign/installation/\n")
    return nil // soft-fail
}
```

### Étape 4 — Ordre de vérification

1. Télécharger `checksums.txt`, `.sig`, `.pem`
2. Vérifier Cosign → si échec, **refuser la mise à jour**
3. Télécharger l'archive
4. Vérifier SHA256 de l'archive contre `checksums.txt` → si échec, refuser
5. Extraire et installer

## Tests

| Scénario | Cosign | SHA256 | Résultat |
|---|---|---|---|
| Tout valide | ✅ | ✅ | Mise à jour OK |
| Signature invalide | ❌ | ✅ | **Refusé** |
| SHA256 mismatch | ✅ | ❌ | **Refusé** |
| Archive + checksums malveillants (même serveur) | ❌ (faux .sig) | ✅ (cohérent) | **Refusé** (Cosign d'abord) |
| cosign absent, MULTIAI_REQUIRE_COSIGN=1 | N/A | ✅ | Erreur |
| cosign absent, pas de REQUIRE | N/A | ✅ | Warning + continue |
