# Story 1.4 — Ajouter un gate CI dans le workflow de release

**Sprint :** 🔴 1 — BLOCKERS
**Priorité :** CRITICAL
**Effort :** 1h
**Agent assigné :** Forge (Dev) + Nexus (Ops)
**Dépendances :** S1.5 (gitleaks), S1.6 (smoke fix)
**Fichier cible :** `multiai-go/.github/workflows/release.yml`

---

## Contexte

Tout tag `v*` déclenche GoReleaser sans aucun test préalable. Une release cassée ou malveillante serait signée Cosign, attestée GitHub, et distribuée via l'auto-update.

Source : Codex adversarial high finding.

## Ce qu'il faut faire

### Étape 1 — Ajouter les checks pré-release

Dans `release.yml`, **avant** l'étape `goreleaser` :

```yaml
jobs:
  preflight:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@<PINNED_SHA>
      
      - name: Verify tag is on master
        run: |
          git fetch origin master
          git merge-base --is-ancestor ${{ github.ref }} origin/master || {
            echo "ERROR: Tag is not a descendant of master"
            exit 1
          }
      
      - uses: actions/setup-go@<PINNED_SHA>
        with:
          go-version: '1.22'
      
      - name: Run tests
        run: |
          cd multiai-go
          go test -race ./...
      
      - name: Run vet
        run: |
          cd multiai-go
          go vet ./...
      
      - name: Run vulncheck
        run: |
          cd multiai-go
          go run golang.org/x/vuln/cmd/govulncheck@v1.0.4 ./...
      
      - name: Run gitleaks
        uses: gitleaks/gitleaks-action@v2
        with:
          config-path: .gitleaks.toml
  
  release:
    needs: preflight
    # ... suite inchangée
```

### Étape 2 — Protéger l'environnement de release

Dans GitHub → Settings → Environments :
- Créer un environnement `release`
- Ajouter une règle de protection : `Required reviewers` ≥ 1
- Référencer l'environnement dans le job `release` :
  ```yaml
  release:
    environment: release
    needs: preflight
  ```

### Étape 3 — Tester

```bash
# Simuler : tag sur une branche non-ancêtre de master
git checkout -b test-release
git tag v9.9.9-test
git push origin v9.9.9-test
# → Le job preflight doit échouer
# → Le job release ne doit PAS s'exécuter
git tag -d v9.9.9-test
git push --delete origin v9.9.9-test
```

## Tests

- [ ] Push d'un tag non descendant de master → preflight échoue
- [ ] Push d'un tag avec tests qui échouent → preflight échoue
- [ ] Push d'un tag valide sur master, CI verte → release OK
- [ ] gitleaks trouve un secret → release bloquée
- [ ] govulncheck trouve une CVE → release bloquée

## Résultat attendu

```
✅ preflight obligatoire avant GoReleaser
✅ Tests, vet, vulncheck, gitleaks exécutés
✅ Tag non-ancêtre de master rejeté
✅ Environnement protégé avec reviewer requis
```

## Sortie de story

- `release.yml` modifié avec job `preflight`
- Environnement GitHub `release` protégé
- Impossible de publier sans CI verte
