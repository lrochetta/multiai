# Story 1.2 — Hardcoder l'URL de l'API GitHub dans l'auto-update

**Sprint :** 🔴 1 — BLOCKERS
**Priorité :** CRITICAL
**Effort :** 30 min
**Agent assigné :** Forge (Dev)
**Dépendances :** Aucune
**Fichier cible :** `multiai-go/internal/update/update.go`

---

## Contexte

L'auto-update utilise `os.Getenv("MULTIAI_GITHUB_API_URL")` sans validation. N'importe qui peut pointer ça vers un serveur malveillant → RCE.

Source : Audit sécurité C-2 + Codex adversarial critical finding.

## Ce qu'il faut faire

### Étape 1 — Modifier `FetchLatestRelease()`

Dans `multiai-go/internal/update/update.go`, fonction `FetchLatestRelease()` :

**Avant (vulnérable) :**
```go
apiURL := os.Getenv("MULTIAI_GITHUB_API_URL")
if apiURL == "" {
    apiURL = fmt.Sprintf("https://api.github.com/repos/%s/releases/latest", repo)
}
```

**Après (sécurisé) :**
```go
apiURL := "https://api.github.com/repos/lrochetta/multiai/releases/latest"
if devURL := os.Getenv("MULTIAI_GITHUB_API_URL"); devURL != "" {
    if os.Getenv("MULTIAI_DEV") != "1" {
        return nil, fmt.Errorf("MULTIAI_GITHUB_API_URL requires MULTIAI_DEV=1")
    }
    if !strings.HasPrefix(devURL, "https://api.github.com/") {
        return nil, fmt.Errorf("MULTIAI_GITHUB_API_URL must start with https://api.github.com/")
    }
    apiURL = devURL
}
```

### Étape 2 — Mettre à jour le commentaire de fonction

Documenter le changement de comportement dans le commentaire godoc de `FetchLatestRelease`.

### Étape 3 — Exécuter les tests

```bash
cd multiai-go

```

## Tests manuels

| Scénario | Commande | Résultat attendu |
|---|---|---|
| Sans variable | `multiai` (normal) | Utilise l'URL hardcodée |
| URL custom sans DEV | `MULTIAI_GITHUB_API_URL=https://evil.com/api multiai` | Erreur : "requires MULTIAI_DEV=1" |
| URL non-GitHub avec DEV | `MULTIAI_DEV=1 MULTIAI_GITHUB_API_URL=https://evil.com/api multiai` | Erreur : "must start with https://api.github.com/" |
| URL GitHub custom avec DEV | `MULTIAI_DEV=1 MULTIAI_GITHUB_API_URL=https://api.github.com/repos/test/releases/latest multiai` | OK, utilise l'URL custom |

## Objectif

Impossible de détourner l'auto-update vers un serveur non-GitHub en production. Mode dev préservé pour les tests.

## Code de vérification

```bash
# Vérifier qu'aucune autre variable utilisateur n'est injectée dans des URLs sans validation
cd multiai-go
rg -n 'os\.Getenv.*URL' --glob '*.go' --glob '!*_test.go'
```

## Sortie de story

- `MULTIAI_GITHUB_API_URL` inutilisable en production
- Restriction `api.github.com` uniquement
- Tests unitaires passent
