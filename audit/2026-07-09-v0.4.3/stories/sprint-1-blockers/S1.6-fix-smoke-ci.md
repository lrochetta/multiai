# Story 1.6 — Réparer le job smoke de la CI

**Sprint :** 🔴 1 — BLOCKERS
**Priorité :** CRITICAL
**Effort :** 5 min
**Agent assigné :** Forge (Dev)
**Dépendances :** Aucune
**Fichier cible :** `multiai-go/.github/workflows/ci.yml`

---

## Contexte

Le job `smoke` a `working-directory: multiai-go` mais exécute `cd multiai-go`. Il cherche `multiai-go/multiai-go` qui n'existe pas → le build échoue silencieusement. Le workflow de release contourne entièrement ce job cassé.

Source : Codex adversarial medium finding.

## Ce qu'il faut faire

### Étape 1 — Supprimer le `cd` redondant

Dans `ci.yml`, job `smoke` :

**Avant :**
```yaml
smoke:
  runs-on: ${{ matrix.os }}
  defaults:
    run:
      working-directory: multiai-go
  steps:
    - run: cd multiai-go && go build -o multiai ./cmd/multiai
    - run: ./multiai version
```

**Après :**
```yaml
smoke:
  runs-on: ${{ matrix.os }}
  defaults:
    run:
      working-directory: multiai-go
  steps:
    - run: go build -o multiai ./cmd/multiai
    - run: ./multiai version
    - run: ./multiai list --json
```

### Étape 2 — Vérifier que le job passe

```bash
# Localement
cd multiai-go
go build -o multiai.exe ./cmd/multiai
./multiai.exe version
./multiai.exe list --json
```

### Étape 3 — Vérifier dans la CI

Pousser la modification et vérifier que le job `smoke` passe sur les 3 OS (ubuntu, macos, windows).

## Tests
