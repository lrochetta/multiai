# Story 1.5 — Installer gitleaks + pre-commit hook + job CI

**Sprint :** 🔴 1 — BLOCKERS
**Priorité :** CRITICAL
**Effort :** 1h
**Agent assigné :** Sentinel (QA/Sécurité)
**Dépendances :** S1.1 (rotation clé DeepSeek avant scan)
**Fichiers :** `.gitleaks.toml`, `.github/workflows/ci.yml`

---

## Contexte

Aucun détecteur automatique de secrets n'est en place. La clé DeepSeek est restée sur le disque sans détection. Aucune protection dans la CI ou en pre-commit.

Source : Audit sécurité H-5.

## Ce qu'il faut faire

### Étape 1 — Créer `.gitleaks.toml`

```toml
title = "multiai gitleaks config"

[allowlist]
  description = "Placeholders et tests"
  paths = [
    '''audit/''',
    '''\.git/''',
  ]
  regexes = [
    '''PASTE_.*_HERE''',
    '''sk-test-''',
    '''-wizard-test-''',
    '''__MULTIAI_CREDSTORE__''',
  ]

[[rules]]
  id = "openai-api-key"
  description = "OpenAI API Key"
  regex = '''sk-proj-[A-Za-z0-9_-]{20,}'''
  
[[rules]]
  id = "anthropic-api-key"
  description = "Anthropic API Key"
  regex = '''sk-ant-[A-Za-z0-9_-]{20,}'''
  
[[rules]]
  id = "deepseek-api-key"
  description = "DeepSeek API Key"
  regex = '''sk-[a-z0-9]{32}'''
  
[[rules]]
  id = "github-pat"
  description = "GitHub Personal Access Token"
  regex = '''ghp_[A-Za-z0-9]{36}'''
  
[[rules]]
  id = "huggingface-key"
  description = "HuggingFace API Key"
  regex = '''hf_[A-Za-z0-9]{34}'''
```

### Étape 2 — Ajouter le job CI

Dans `ci.yml` :

```yaml
  secret-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@<PINNED_SHA>
        with:
          fetch-depth: 0
      - uses: gitleaks/gitleaks-action@v2
        with:
          config-path: .gitleaks.toml
```

### Étape 3 — Configurer le pre-commit hook (optionnel)

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/gitleaks/gitleaks
    rev: v8.18.0
    hooks:
      - id: gitleaks
```

### Étape 4 — Premier scan

```bash
gitleaks detect --config .gitleaks.toml --verbose
```

Corriger les éventuels findings (faux positifs dans les tests → ajouter à l'allowlist).
