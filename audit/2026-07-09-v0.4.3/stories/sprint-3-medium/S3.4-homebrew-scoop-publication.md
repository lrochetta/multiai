# Story 3.4 — Homebrew tap + Scoop bucket (`skip_upload: false`)

**Sprint :** 🟡 3 — MEDIUM
**Priorité :** MEDIUM
**Effort :** 3h
**Agent assigné :** Nexus (Ops)
**Dépendances :** S1.4 (gate CI avant distribution)
**Fichier cible :** `.goreleaser.yaml`

---

## Objectif

Les canaux de distribution Homebrew et Scoop sont configurés dans `.goreleaser.yaml` mais avec `skip_upload: true`. Les rendre opérationnels.

## Ce qu'il faut faire

### Étape 1 — Créer les repos GitHub

```bash
# Créer le Homebrew tap
gh repo create lrochetta/homebrew-tap --public --description "Homebrew tap for multiai and other tools"

# Créer le Scoop bucket
gh repo create lrochetta/scoop-bucket --public --description "Scoop bucket for multiai and other tools"
```

### Étape 2 — Créer un token GitHub

1. Aller sur https://github.com/settings/tokens
2. Créer un **classic PAT** avec le scope `public_repo`
3. Nom : `MULTIAI_TAP_TOKEN`
4. Copier le token

### Étape 3 — Ajouter le secret dans le repo

```bash
gh secret set TAP_GITHUB_TOKEN --body "ghp_..." --repo lrochetta/multiai
```

### Étape 4 — Modifier `.goreleaser.yaml`

```yaml
homebrew_casks:
  - name: multiai
    repository:
      owner: lrochetta
      name: homebrew-tap
      token: "{{ .Env.TAP_GITHUB_TOKEN }}"
    skip_upload: false  # ← changer

scoops:
  - name: multiai
    repository:
      owner: lrochetta
      name: scoop-bucket
      token: "{{ .Env.TAP_GITHUB_TOKEN }}"
    skip_upload: false  # ← changer
```

### Étape 5 — Tester en local

```bash
cd multiai-go
goreleaser release --snapshot --clean
# Vérifier les fichiers dans dist/homebrew/ et dist/scoop/
brew audit --strict dist/homebrew/Casks/multiai.rb  # macOS
```

### Étape 6 — Valider après la première release

```bash
# macOS
brew tap lrochetta/homebrew-tap
brew install multiai

# Windows
scoop bucket add multiai https://github.com/lrochetta/scoop-bucket
scoop install multiai
```
