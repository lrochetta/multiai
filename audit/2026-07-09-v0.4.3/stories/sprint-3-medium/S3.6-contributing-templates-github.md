## Objectif

Faciliter les contributions externes. Actuellement : pas de CONTRIBUTING.md, pas de templates d'issues, pas de guide de PR.

## Ce qu'il faut faire

### Fichier 1 : `CONTRIBUTING.md` (racine du repo)

```markdown
# Contribuer à multiai

## Environnement de développement

- Go 1.22+
- `go build ./cmd/multiai`
- `go test ./...`

## Conventions

- **Commits** : Conventional Commits (`feat:`, `fix:`, `docs:`, `chore:`)
- **Code** : `gofmt`, `go vet`, `golangci-lint run`
- **Tests** : obligatoires pour toute nouvelle fonctionnalité

## Processus

1. Fork → branche → PR
2. La CI doit être verte
3. Un reviewer approuve
4. Merge sur master

## Guide de release

1. `git tag vX.Y.Z`
2. Push → CI release
3. Vérifier GitHub Releases
4. `npm publish` (manuel)

## Questions ?

Ouvrez une issue ou contactez [@lrochetta](https://github.com/lrochetta)
```
