# Story 3.5 — SBOM CycloneDX dans les releases

**Sprint :** 🟡 3 — MEDIUM
**Priorité :** MEDIUM
**Effort :** 1h
**Agent assigné :** Nexus (Ops)
**Dépendances :** Aucune
**Fichier cible :** `.github/workflows/release.yml`

---

## Objectif

Générer un SBOM au format CycloneDX JSON et le publier avec chaque release.

## Ce qu'il faut faire

### Étape 1 — Ajouter l'étape SBOM dans release.yml

```yaml
- name: Generate SBOM
  uses: anchore/sbom-action@v0
  with:
    path: ./multiai-go
    format: cyclonedx-json
    output-file: sbom.cyclonedx.json

- name: Upload SBOM to release
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
  run: |
    gh release upload ${{ github.ref_name }} sbom.cyclonedx.json \
      --repo ${{ github.repository }} \
      --clobber
```

### Étape 2 — Signer le SBOM avec Cosign (optionnel)

```yaml
- name: Sign SBOM
  run: |
    cosign sign-blob \
      --yes \
      --output-signature sbom.cyclonedx.json.sig \
      --output-certificate sbom.cyclonedx.json.pem \
      sbom.cyclonedx.json
- name: Upload SBOM signature
  run: |
    gh release upload ${{ github.ref_name }} \
      sbom.cyclonedx.json.sig \
      sbom.cyclonedx.json.pem
```

### Étape 3 — Vérifier

```bash
# Après la release
gh release view v0.5.0 --repo lrochetta/multiai
# Doit lister sbom.cyclonedx.json dans les assets
```

## Résultat attendu

```
✅ SBOM généré à chaque release
✅ Format CycloneDX JSON
✅ Inclus dans les assets de la release GitHub
✅ (Optionnel) Signé avec Cosign
```
