## Contexte

`ServiceForProfile` dérive le nom du service du **basename** du fichier profil uniquement. Un attaquant peut créer `/tmp/evil/00-claude-official.env` qui récupère le secret de `~/.multiai/profiles/00-claude-official.env`.

Source : Codex adversarial high finding.

## Ce qu'il faut faire

### Étape 1 — Inclure le chemin canonique dans le nom du service

```go
func ServiceForProfile(profilePath string) string {
    abs, err := filepath.Abs(profilePath)
    if err != nil {
        abs = profilePath
    }
    canonical := filepath.Clean(abs)
    
    // Hash des 8 premiers octets du SHA256 du chemin canonique
    h := sha256.Sum256([]byte(canonical))
    hashSuffix := fmt.Sprintf("%x", h[:8])
    
    base := strings.TrimSuffix(filepath.Base(canonical), filepath.Ext(canonical))
    return fmt.Sprintf("multiai/%s-%s", base, hashSuffix)
}
```

### Étape 2 — Migration automatique des anciens noms

```go
func migrateServiceName(store Store, oldName, newName string) error {
    secrets, err := store.List(oldName)
    if err != nil || len(secrets) == 0 {
        return nil // rien à migrer
    }
    for k, v := range secrets {
        if err := store.Set(newName, k, v); err != nil {
            return err
        }
        store.Delete(oldName, k)
    }
    return nil
}
```

Appeler `migrateServiceName` au premier `Get` si l'ancien nom existe.
