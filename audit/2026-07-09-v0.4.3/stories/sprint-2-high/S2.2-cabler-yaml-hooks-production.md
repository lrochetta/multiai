# Story 2.2 — Câbler YAML + hooks + config projet dans le chemin de production

**Sprint :** 🟠 2 — HIGH
**Priorité :** HIGH
**Effort :** 3h
**Agent assigné :** Forge (Dev)
**Dépendances :** S2.1 (tests avant refactoring)
**Fichiers :** `multiai-go/cmd/multiai/main.go`, `multiai-go/internal/cli/launcher.go`

---

## Contexte

Les profils `.yaml`/`.yml`, la config projet `.multiai.yaml` et les hooks `before_launch`/`after_launch` sont **implémentés mais jamais appelés en production**.

`main.go` utilise `profile.LoadDir()` qui ne charge que les `.env`. Résultat : fonctionnalités documentées dans le README mais absentes à l'exécution.

Source : Codex adversarial high finding.

## Ce qu'il faut faire

### Étape 1 — Remplacer `LoadDir` par `LoadAllProfiles`

Dans `main.go`, fonction `runLaunch()` et fonction `listProfiles()` :

```go
// Avant
profiles, err := profile.LoadDir(profilesDir)

// Après
profiles, err := profile.LoadAllProfiles(profilesDir)
```

### Étape 2 — Charger la config projet

```go
// Après LoadAllProfiles
projectConfig, err := profile.FindProjectConfig()
if err == nil && projectConfig != nil {
    for i := range profiles {
        profile.MergeProjectConfig(&profiles[i], projectConfig)
    }
}
```

### Étape 3 — Transmettre les hooks à LaunchOptions

```go
opts := cli.LaunchOptions{
    // ... existant
    BeforeHooks: prof.BeforeHooks,
    AfterHooks:  prof.AfterHooks,
}
```

### Étape 4 — Activer l'exécution des hooks

Dans `runLaunch()`, décommenter ou ajouter :

```go
// Avant le lancement
if len(opts.BeforeHooks) > 0 && !opts.DryRun {
    if os.Getenv("MULTIAI_NO_HOOKS") != "1" {
        if err := cli.RunBeforeHooks(opts.BeforeHooks, prof); err != nil {
            cli.PrintWarning("Hook before_launch a échoué : %v", err)
        }
    }
}

// ... lancer le processus enfant ...

// Après le lancement
if len(opts.AfterHooks) > 0 && !opts.DryRun {
    if os.Getenv("MULTIAI_NO_HOOKS") != "1" {
        cli.RunAfterHooks(opts.AfterHooks, prof) // erreur ignorée volontairement
    }
}
```

### Étape 5 — Ajouter un flag `--no-hooks`

Dans `main.go`, ajouter le support du flag `--no-hooks` (déjà supporté via `MULTIAI_NO_HOOKS`).

## Tests manuels

### Test 1 : Profil YAML
```bash
cat > ~/.multiai/profiles/test-yaml.yaml << 'EOF'
id: test-yaml
tool: claude
display_name: "Test YAML"
command: claude
env:
  ANTHROPIC_API_KEY: PASTE_YOUR_KEY_HERE
EOF
multiai list | grep "test-yaml"
multiai launch -p test-yaml --dry-run
```

### Test 2 : Hook before_launch
```bash
cat > ~/.multiai/profiles/test-hook.env << 'EOF'
PROFILE_ID=test-hook
TOOL=claude
COMMAND=echo
BEFORE_LAUNCH=echo "Hook exécuté avant lancement"
EOF
multiai launch -p test-hook 2>&1 | grep "Hook exécuté"
```

### Test 3 : Config projet
```bash
cat > .multiai.yaml << 'EOF'
profiles:
  ds:
    overrides:
      CLEAR_ENV: "false"
EOF
multiai launch -p ds --show-env | grep CLEAR_ENV
```

### Test 4 : Pas de hooks avec MULTIAI_NO_HOOKS
```bash
MULTIAI_NO_HOOKS=1 multiai launch -p test-hook
# Le hook NE doit PAS s'exécuter
```
