# Story 2.5 — Transactionalité store/sentinelle (race condition)

**Sprint :** 🟠 2 — HIGH
**Priorité :** HIGH
**Effort :** 4h
**Agent assigné :** Forge (Dev)
**Dépendances :** S2.3 (ServiceForProfile avant transactionnalité)
**Fichiers :** `multiai-go/internal/secret/`, `multiai-go/internal/config/wizard.go`

---

## Contexte

- Deux processus multiai simultanés peuvent corrompre le store : lecture + écriture non atomiques
- Le wizard efface le secret AVANT de remplacer la sentinelle → crash = perte irrécupérable
- `encryptedFileStore` lit tout le fichier, modifie une entrée, réécrit tout → deux processus s'écrasent

Source : Codex adversarial high finding.

## Ce qu'il faut faire

### Étape 1 — Une entrée par clé (fichier par secret)

Remplacer le fichier unique `secrets.enc` par un dossier `secrets/` :

```
~/.config/multiai/secrets/
├── ds-a1b2c3d4/
│   ├── DEEPSEEK_API_KEY.enc
│   └── .lock
├── claude-a1b2c3d4/
│   ├── ANTHROPIC_API_KEY.enc
│   └── .lock
```

Chaque clé est chiffrée individuellement → deux processus modifiant des clés différentes ne se bloquent pas.

### Étape 2 — Verrou inter-processus (flock / LockFileEx)

```go
type lockedFile struct {
    *os.File
}

func lockFile(path string) (*lockedFile, error) {
    f, err := os.OpenFile(path, os.O_CREATE|os.O_RDWR, 0600)
    if err != nil {
        return nil, err
    }
    if err := syscall.Flock(int(f.Fd()), syscall.LOCK_EX); err != nil {
        f.Close()
        return nil, err
    }
    return &lockedFile{f}, nil
}

func (lf *lockedFile) unlock() {
    syscall.Flock(int(lf.Fd()), syscall.LOCK_UN)
    lf.Close()
}
```

### Étape 3 — Transaction store + profil

Dans `config/wizard.go`, fonction `updateEnvFile` :

```go
func updateEnvFile(profilePath string, updates map[string]string) error {
    // 1. D'abord, écrire tous les secrets dans le store (avant de toucher au .env)
    store := getStore()
    for key, value := range updates {
        if value != secret.Sentinel {
            if err := store.Set(serviceName, key, value); err != nil {
                // Store échoue → ne pas toucher au .env
                return fmt.Errorf("store write failed: %w", err)
            }
        }
    }
    
    // 2. Ensuite, écrire les sentinelles dans le .env (atomique)
    if err := writeEnvFile(profilePath, sentinelUpdates); err != nil {
        // .env échoue → rollback du store
        for key := range sentinelUpdates {
            store.Delete(serviceName, key)
        }
        return fmt.Errorf("env write failed, rolled back: %w", err)
    }
    
    return nil
}
```

### Étape 4 — Ne jamais effacer avant d'avoir écrit

Règle d'or : l'ordre doit être **CREATE NEW → REPLACE OLD → DELETE OLD**

```go
// ❌ MAUVAIS (existant)
store.Delete(service, key)  // efface d'abord
store.Set(service, key, newValue)  // écrit ensuite → crash entre les deux = perte

// ✅ BON
store.Set(service, key+".new", newValue)  // crée le nouveau
store.Delete(service, key)                // efface l'ancien
store.Set(service, key, newValue)         // écrit au bon nom
store.Delete(service, key+".new")         // nettoie le temporaire
```

## Tests

| Scénario | Résultat attendu |
|---|---|
| Deux `Set` concurrents sur la même clé | Le dernier gagne, pas de corruption |
| Crash entre store.Set et .env write | Rollback → état cohérent |
| Disque plein pendant store.Set | Erreur, .env non modifié |
| Ctrl+C pendant l'écriture | État cohérent au prochain lancement |
| 10 processus simultanés | 0 corruption |

### Test multi-processus

```go
func TestConcurrentStoreAccess(t *testing.T) {
    dir := t.TempDir()
    // Lancer 10 goroutines qui lisent/écrivent simultanément
    var wg sync.WaitGroup
    for i := 0; i < 10; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            store := newStore(dir)
            for j := 0; j < 100; j++ {
                store.Set("test", fmt.Sprintf("key-%d", j), fmt.Sprintf("val-%d-%d", id, j))
                store.Get("test", fmt.Sprintf("key-%d", j))
            }
        }(i)
    }
    wg.Wait()
    // Vérifier l'intégrité
}
```
