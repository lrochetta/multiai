# Story 2.3 — Gérer `--store` explicitement (refuser ou implémenter)

**Sprint :** 🟠 2 — HIGH
**Priorité :** HIGH
**Effort :** 4h (Option A) ou 30 min (Option B)
**Agent assigné :** Forge (Dev)
**Dépendances :** Aucune
**Fichiers :** `multiai-go/cmd/multiai/main.go`, `multiai-go/internal/secret/store_windows.go`

---

## Contexte

`multiai config --store keychain|wincred|secret-service` est accepté mais **ignoré silencieusement**. Les 3 fichiers `store_*.go` sont des stubs identiques. L'utilisateur pense utiliser le store natif, mais utilise le fichier AES sans le savoir.

Source : Codex adversarial high finding.

## Option B : Refus explicite (pragmatique, 30 min)

### Ce qu'il faut faire

1. Dans `main.go`, gérer le flag `--store` :

```go
if storeFlag := getFlagValue(os.Args, "--store"); storeFlag != "" {
    validStores := map[string]bool{
        "keychain": true, "wincred": true, "secret-service": true,
    }
    if !validStores[storeFlag] {
        fmt.Fprintf(os.Stderr, "[X] Store invalide : %s (valides : keychain, wincred, secret-service)\n", storeFlag)
        os.Exit(2)
    }
    fmt.Fprintf(os.Stderr, "[i] Le backend natif '%s' n'est pas encore implémenté.\n", storeFlag)
    fmt.Fprintf(os.Stderr, "    Le store chiffré par fichier (AES-256-GCM) est utilisé.\n")
    fmt.Fprintf(os.Stderr, "    Suivez https://github.com/lrochetta/multiai/issues/42\n")
    // Continuer avec le wizard normal (fichier AES)
}
```

2. Logger le backend utilisé :

```go
func NewStore() (Store, error) {
    store, err := newPlatformStore()
    if err != nil {
        return nil, err
    }
    fmt.Fprintf(os.Stderr, "[i] Credential store : %s\n", store.Name())
    return store, nil
}
```
